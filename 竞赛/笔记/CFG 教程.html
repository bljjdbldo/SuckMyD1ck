1 cfg 的存放位置
绝大部分 cfg 都需要存放在 CS2 的 cfg 文件夹才能被游戏读取。你可以按照如下步骤找到他：
●在 steam 库的左侧列表中，右键 CS2，选择 “管理”，再选择 “浏览本地文件”，打开 CS2 的文件目录
●👉 进入 game 文件夹
●👉 进入 csgo 文件夹
●👉 进入 cfg 文件夹
这里就是存放 cfg 的位置了。
另外，强烈建议你打开 windows 的 “显示文件扩展名功能”，否则容易发生 a.cfg.txt 这样的错误文件名。你可以在网上搜索相关资料，此处不做赘述
2 怎么让游戏读取 cfg
了解如何手动读取 cfg 文件
你需要先学会开启游戏内控制台，这会是你 cfg 路上的得力助手 🤗 可以在网上搜索相关教程，有很多。
假设 cfg 文件夹内有一个 jiting.cfg 文件，你需要在控制台内输入 exec jiting 或 exec jiting.cfg 即可读取。
.cfg 后缀名可以省略。这是独属于 .cfg 文件的特性。CS2 在你未指定文件后缀名时会默认为 .cfg 文件。
如果读取正常，控制台应该会输出如下信息 😎：
​
9
1
[InputService] execing jiting
一个常见的报错信息是 💀：
​
9
1
[InputService] exec: couldn't exec '{*}cfg/jiting.cfg', unable to read file
这句话的意思是找不到对应的 cfg 文件 😭。也许你应该检查你存放 cfg 的文件夹是否正确，并检查输入的文件名是否正确 🤔
我们暂时还用不到这种独立的 cfg 文件。不过掌握这个是非常重要的。
如何让 cfg 文件在游戏启动时自动读取
●使用 steam 启动项
在 steam 中右键 CS2，点击属性，可以看到一个输入启动选项的窗口。在这里输入 +exec 文件名 或 +exec 目录/文件名 即可。不同的启动项之间要加空格. 注意启动项在 exec 前有一个 +。
例如在启动项中填写 +exec jiting.cfg 来自动加载上文提到的 cfg 文件.
●使用 autoexec.cfg 文件
在 cfg 文件夹中找到 autoexec.cfg 文件（如果没有可以自行新建一个）。这个文件会在游戏启动时被自动读取，你可以在里面书写你的 cfg，不过这并不推荐。一个更好的做法是在里面使用 exec 来加载你自己另外的的 cfg 文件。
例如在 autoexec.cfg 中另起一行，填写 exec jiting.cfg 来自动加载上文提到的 CFG 文件.
5E 需要去 5E 平台的 5ebox 内额外配置启动项。完美平台会自动使用 steam 内的启动项。
3 简单的指令
CS2 中的大部分操作都是由 cfg 驱动的，绝大部分的动作都可以用 cfg 完成。
单行指令可以直接在控制台输入。如果有多行指令需要一起触发，就需要把它们保存进 .cfg 文件，用上面说的方法读取。
我们都知道，平台的发刀方法是在聊天区输入 .drop。该怎么用 cfg 来完成这一点呢？

但这样，每次我们都得手动打开控制台输入，还不如直接发消息呢。怎么把它绑定到一个按键上呢？

关闭控制台，按 p，发送了 .drop ！于是现在可以一键发刀了！😎 你可以把 p 改成其他按键
bind：把指令绑定给一个键。使用格式为 bind 你要绑定的键 要绑定的指令 bind 是永久性的，除非再次 bind 这个键。也就是说，在控制台 bind 一次后，重启游戏也不会失效 👍

实际上是这样的：attack x 0 0 给攻击的指令计数 +x。例如 attack 5 0 0 将会给攻击指令计数 +5。attack -3 0 0 将会给攻击指令计数 -3。
当指令的指令计数大于 0 时，指令就会生效。对于 attack，就是会开火。
指令计数不能小于 0。小于 0 会强制变成 0。例如，假如此时的攻击指令计数为 1，你正在开火，执行 attack -9 0 0 不会让它变成 -8，而是变成 0
你可以理解为：
●+attack 只是对于 attack 1 0 0 的一个 alias
●-attack 只是对于 attack -1 0 0 的一个 alias

键盘怎么打出这个字符？😥

简单来说，它可以把左侧的输出传递给右侧的指令，作为输入。
